# Glossary

Canonical terminology for the Lone Wolf CYOA project.

| Term | Definition |
|------|-----------|
| **Section** | A numbered passage in a book. The fundamental unit of gameplay. Sections contain narrative text, choices, combat encounters, and item events. |
| **Choice** | A player-selectable option within a section that leads to another section. May be gated by a condition (discipline, item, gold). |
| **Discipline** | A special ability chosen at character creation. Disciplines unlock gated choices, provide combat bonuses, or grant passive effects (e.g., Healing, Hunting). |
| **Era** | A grouping of books with shared mechanics and discipline sets: Kai (1–5), Magnakai (6–12), Grand Master (13–20), New Order (21–28). |
| **Combat Skill (CS)** | A character's base fighting ability. Modified by disciplines, weapons, and encounter-specific rules. |
| **Endurance (END)** | A character's health. Reaches 0 = death. |
| **Combat Ratio (CR)** | `hero_effective_cs - enemy_cs`. Used to index into the Combat Results Table. |
| **Combat Results Table (CRT)** | A lookup table mapping (combat ratio bracket, random number) → (enemy loss, hero loss). 13 brackets × 10 random numbers. |
| **Run** | A single playthrough attempt of a book. Run 1 is the first attempt. When a character dies, they restart at run N+1. |
| **Snapshot** | A saved copy of character state at the beginning of a book. Used to restore the character on death. |
| **Source (column)** | Tracks whether a content value was auto-generated by the parser (`auto`) or manually set by an admin (`manual`). Parser re-runs only overwrite `auto` values. |
| **Display text** | The player-facing version of choice text, rewritten by Haiku to remove "turn to page X" phrasing. |
| **Lore-circle** | A grouping of Magnakai+ disciplines that grants stat bonuses when all disciplines in the circle are learned. |
| **Psi-surge** | A Magnakai discipline. Opt-in per combat round: +4 CS but costs 2 END per round. |
| **Evasion** | Some combat encounters allow the character to flee after N rounds, redirecting to a different section. |
| **Special item** | An item that cannot be voluntarily dropped and doesn't count toward backpack limits. Often story-critical. |
| **Gold Crowns** | Currency. Carried in belt pouch. Maximum 50. |
| **Meal** | A backpack item consumed when instructed to eat. The Hunting discipline negates meal requirements. |
| **Wizard (book advance)** | The multi-step process for transitioning a character from one book to the next: discipline selection, inventory adjustment, stat carry-over. |
| **Report** | A player-submitted bug report tied to a specific section. Tagged with categories and optionally free text. Triaged by admins. |
| **Admin** | A separate user account (in `admin_users` table) with access to content management and report triage endpoints. |
| **Phase** | A discrete step in a section's processing sequence (items, eat, combat, heal, choices). The character must resolve each phase in order before making choices. |
| **Phase sequence** | The ordered list of phases for a section. Computed at runtime from section properties, or overridden per-section via `phase_sequence_override`. |
| **Character event** | A logged state change in the `character_events` table. One row per phase step completion, with event type and JSON details. Provides a full audit trail of all mutations. |
| **Weapon category** | A grouping of weapon names (e.g., "Sword" includes Sword, Broadsword, Short Sword). Used for Weaponskill/Weaponmastery bonus matching instead of exact name comparison. |
| **World entity** | A named thing in the Lone Wolf universe: a character, location, creature, or organization. Global across all books, identified by canonical name with aliases. |
| **Appearance** | A record linking a world entity to a specific section, with a role (combatant, mentioned, visited, etc.) and context snippet. Tracks every section an entity appears in, including narrative references. |
| **Relationship (entity)** | A typed, directed link between two world entities. Categorized as social, spatial, factional, temporal, or causal, with a freeform type within each category. |
| **Entity extraction** | The LLM-powered step in the parser pipeline that identifies world entities in narrative text, deduplicates against the existing catalog, and infers relationships. |
| **Knowledge graph** | The combined set of world entities, their appearances across sections/books, and their relationships. Forms a queryable taxonomy of the Lone Wolf universe. |
| **Random outcome** | One band within a phase-based random roll. Maps a number range (e.g., 0–4) to an effect (gold change, END change, item gain/loss, section redirect). Stored in the `random_outcomes` table. Distinct from choice-based random branching. |
| **Rule override** | A per-character JSON configuration that controls rule variant behavior (e.g., discipline stacking mode). Stored in `characters.rule_overrides`. Allows different characters to use different rule interpretations. |
| **Soft delete** | Characters are never permanently removed. `DELETE /characters/{id}` sets `is_deleted = true` and `deleted_at`. Character is hidden from player lists but preserved for analytics. Admin can restore. |
| **Replay** | Restarting the current book from the beginning after reaching a victory section, as an alternative to advancing to the next book. Resets to `character_book_starts` snapshot, increments `current_run` but not `death_count`. |
| **Conditional combat** | A combat encounter that only triggers when the character lacks a specific discipline or item. Modeled via `condition_type`/`condition_value` on `combat_encounters`. |
| **LLM cache** | Local cache of parser LLM results (choice rewrites, entity extractions). Keyed by input text hash. Avoids redundant API calls on re-runs. Stored in `.parser_cache/`. |
