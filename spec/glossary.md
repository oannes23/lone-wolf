# Glossary

Canonical terminology for the Lone Wolf CYOA project.

| Term | Definition |
|------|-----------|
| **Scene** | A numbered passage in a book. The fundamental unit of gameplay. Scenes contain narrative text, choices, combat encounters, and item events. Corresponds to a numbered "section" in the original Lone Wolf books. Each scene has a dual representation: a `scenes` row for gameplay data and a `game_objects` entry (kind='scene') for taxonomy. |
| **Choice** | A player-selectable option within a scene that leads to another scene. May be gated by a condition (discipline, item, gold). Compound conditions supported via JSON `condition_value` (e.g., `{"any": ["Tracking", "Huntmastery"]}`). |
| **Discipline** | A special ability chosen at character creation. Disciplines unlock gated choices, provide combat bonuses, or grant passive effects (e.g., Healing, Hunting). |
| **Era** | A grouping of books with shared mechanics and discipline sets: Kai (1–5), Magnakai (6–12), Grand Master (13–20), New Order (21–28). |
| **Combat Skill (CS)** | A character's base fighting ability. Modified by disciplines, weapons, and encounter-specific rules. |
| **Endurance (END)** | A character's health. Reaches 0 = death. |
| **Endurance Max** | The maximum endurance a character can have, tracked separately from `endurance_base`. Increased by permanent bonuses like lore-circle effects. Healing caps at this value, not at base. |
| **Combat Ratio (CR)** | `hero_effective_cs - enemy_cs`. Used to index into the Combat Results Table. |
| **Combat Results Table (CRT)** | A lookup table mapping (combat ratio bracket, random number) → (enemy loss, hero loss). 13 brackets × 10 random numbers. |
| **Run** | A single playthrough attempt of a book. Run 1 is the first attempt. When a character dies, they restart at run N+1. |
| **Snapshot** | A saved copy of character state at the beginning of a book. Used to restore the character on death. |
| **Source (column)** | Tracks whether a content value was auto-generated by the parser (`auto`) or manually set by an admin (`manual`). Parser re-runs only overwrite `auto` values. |
| **Display text** | The player-facing version of choice text, rewritten by Haiku to remove "turn to page X" phrasing. |
| **Lore-circle** | A grouping of Magnakai+ disciplines that grants stat bonuses (including `endurance_max` increases) when all disciplines in the circle are learned. |
| **Psi-surge** | A Magnakai discipline. Opt-in per combat round: +4 CS but costs 2 END per round. |
| **Evasion** | Some combat encounters allow the character to flee after N rounds, redirecting to a different scene. Evasion damage is configurable per encounter (default 0). |
| **Special item** | An item that cannot be voluntarily dropped and doesn't count toward backpack limits. Often story-critical. |
| **Gold Crowns** | Currency. Carried in belt pouch. Maximum 50. On pickup, partial acceptance up to cap (auto-applied). |
| **Meal** | Tracked as an integer counter on the character (not as inventory items). Maximum 8 (matches backpack capacity thematically). On pickup, partial acceptance up to cap. The Hunting discipline negates meal requirements. Meals do not count against the backpack limit. |
| **Game Object** | Any entity in the game world tracked by the taxonomy system. Game objects have a Kind, canonical name, optional aliases, and kind-specific properties. All world entities, items, foes, and scenes are game objects. Stored in the `game_objects` table. |
| **Kind** | A game object's type classification: `character`, `location`, `creature`, `organization`, `item`, `foe`, or `scene`. Determines what properties are expected and how the object participates in gameplay. |
| **Tagged Ref** | A typed, directed link between two game objects stored in the `game_object_refs` table. Uses a JSON tags array to encode relationship category and type (e.g., `["appearance", "combatant"]`, `["spatial", "located_in"]`). Replaces separate appearances and relationships tables. |
| **Wizard Template** | A data-driven definition of a multi-step wizard. Defines an ordered sequence of step types (stat_roll, pick_disciplines, pick_equipment, etc.) with configuration. Used for both character creation and book advance. |
| **Wizard Progress** | Tracks a character's position within an active wizard, including the current step index and accumulated state (selected disciplines, equipment, etc.). Stored in `character_wizard_progress`. |
| **Optimistic Lock** | Concurrency control via a `version` counter on the character. State-mutating API requests include the expected version; if it doesn't match (another request modified state), the server returns 409. |
| **Backpack Loss** | A scene flag (`loses_backpack`) that causes removal of all backpack items and meals when the character enters the scene. |
| **Report** | A player-submitted bug report tied to a specific scene. Tagged with categories and optionally free text. Triaged by admins. |
| **Admin** | A separate user account (in `admin_users` table) with access to content management and report triage endpoints. First admin created via CLI command. |
| **Phase** | A discrete step in a scene's processing sequence (items, eat, combat, heal, choices). The character must resolve each phase in order before making choices. |
| **Phase sequence** | The ordered list of phases for a scene. Computed at runtime from scene properties, or overridden per-scene via `phase_sequence_override`. |
| **Character event** | A logged state change in the `character_events` table. One row per phase step completion, with event type and JSON details. Provides a full audit trail of all mutations. |
| **Weapon category** | A grouping of weapon names (e.g., "Sword" includes Sword, Broadsword, Short Sword). Used for Weaponskill/Weaponmastery bonus matching instead of exact name comparison. |
| **Entity extraction** | The LLM-powered step in the parser pipeline that identifies game objects in narrative text, deduplicates against the existing catalog, and infers relationships via tagged refs. |
| **Knowledge graph** | The combined set of game objects, their tagged refs across scenes/books, and their inter-object relationships. Forms a queryable taxonomy of the Lone Wolf universe. |
| **Random outcome** | One band within a phase-based random roll. Maps a number range (e.g., 0–4) to an effect (gold change, END change, item gain/loss, scene redirect). Stored in the `random_outcomes` table. Distinct from choice-based random branching. |
| **Rule override** | A per-character JSON configuration that controls rule variant behavior (e.g., discipline stacking mode). Stored in `characters.rule_overrides`. |
| **Soft delete** | Characters are never permanently removed. `DELETE /characters/{id}` sets `is_deleted = true` and `deleted_at`. Character is hidden from player lists but preserved for analytics. Admin can restore. |
| **Replay** | Restarting the current book from the beginning after reaching a victory scene, as an alternative to advancing to the next book. Resets to `character_book_starts` snapshot, increments `current_run` but not `death_count`. |
| **Conditional combat** | A combat encounter that only triggers when the character lacks a specific discipline or item. Modeled via `condition_type`/`condition_value` on `combat_encounters`. |
| **Phase results** | An array on the scene response reporting what happened during automatic phases (eat, heal, item_loss, backpack_loss). Since these phases resolve instantly on the server, the client never sees them as the active phase — only their results. |
| **Choice-triggered random** | A random mechanic where the player first selects a choice, then rolls. The parent choice has `target_scene_id = null` and outcome bands are stored in `choice_random_outcomes`. Distinct from scene-level random exits (all choices random-gated) and phase-based random (background effects from `random_outcomes`). |
| **Scene-level random exit** | A random mechanic where ALL choices in a scene have `condition_type='random'` with number ranges. The player just rolls — no decision is made. The roll determines which scene to navigate to. |
| **LLM cache** | Local cache of parser LLM results (choice rewrites, entity extractions). Keyed by input text hash. Avoids redundant API calls on re-runs. Stored in `.parser_cache/`. |
| **Cascade** | A chain of triggered events where one event's effects cause further events. Scene redirects and death checks are examples. Safety limits (MAX_REDIRECT_DEPTH) prevent infinite loops. Follows ops.md section 4. |
| **Meter** | A bounded numeric field with explicit min/max and defined boundary behavior. Endurance, gold, and meals are meters. When a meter hits its boundary (e.g., endurance reaches 0), a specific game action fires (e.g., death). Follows ops.md section 1.3. |
| **Operation** | An atomic state mutation primitive. Each character_event decomposes into one or more operations (e.g., `meter.delta` for endurance changes, `ref.set` for scene transitions). Eight operations cover all mutations. Follows ops.md section 2.2. |
| **Projection** | A read-optimized view derived from the event log. Character state is currently a mutable projection, not rebuilt from events. Leaderboards are projections from aggregate queries. Follows ops.md section 3. |
| **Spec (field layer)** | A field whose value is authored directly by events — the source of truth. Examples: `combat_skill_base`, `endurance_base`, `name`. Contrast with Status. Follows ops.md section 1.2. |
| **Status (field layer)** | A field whose value is derived/computed from Spec fields — never written directly. Examples: `endurance_max` (from base + lore-circle bonuses), `is_alive` (from endurance > 0). Follows ops.md section 1.2. |
| **Trigger** | A rule that watches for a specific event and automatically fires further effects. Example: any endurance reduction that reaches 0 triggers death. Currently implemented as centralized engine functions, not data-driven rules. Follows ops.md section 4. |
| **Unarmed penalty** | -4 CS penalty applied when fighting without an equipped weapon. Checked in `effective_combat_skill()`. |
| **Enemy Mindblast** | A combat modifier (`modifier_type='enemy_mindblast'`) on encounters where the enemy has psychic attack. Imposes -2 CS on the hero unless they have the Mindshield discipline. |
| **Consumable item** | An item with `consumable: true` in its game_object properties. Can be used at any phase via `POST /gameplay/{id}/use-item`. Consumed (removed from inventory) on use. Effects are data-driven (e.g., `endurance_restore`). |
| **Engine DTO** | A dataclass (`CharacterState`, `SceneContext`, `CombatContext`) that defines the engine's input contract. The API layer populates these from the database; engine functions accept only these types, never ORM models. |
| **Roll group** | An integer on `random_outcomes` that groups outcome bands for scenes requiring multiple sequential rolls. Each group represents one roll; the random phase iterates through groups in order. |
